# RSS订阅阅读器

## 一、题目概述

### 1.1 背景

你正在开发一个**RSS订阅阅读器**。用户可以添加RSS订阅源，查看文章列表，标记已读/未读，收藏文章。

### 1.2 技术要求

- 前端框架：React
- 语言：TypeScript
- 后端：无限制
- 数据存储：本地数据库（自选、无要求）
- 允许并鼓励使用 AI 工具辅助开发

### 1.3 时间限制

**35 分钟**（5分钟沟通 + 20分钟开发完成80%核心功能）

---

## 二、功能需求

### 2.1 订阅源管理

| 优先级 | 功能 | 说明 |
|--------|------|------|
| **P0** | 添加订阅源 | 输入订阅源地址，保存后出现在订阅列表中 |
| **P0** | 订阅源列表 | 展示所有已添加的订阅源（名称、更新时间） |
| **P1** | 刷新订阅源 | 手动刷新某个订阅源，获取最新文章 |
| **P2** | 删除订阅源 | 删除不再需要的订阅源 |

### 2.2 文章列表

| 优先级 | 功能 | 说明 |
|--------|------|------|
| **P0** | 文章列表展示 | 展示所有订阅源的文章（标题、来源、发布时间） |
| **P0** | 文章详情 | 点击文章展示完整内容（或跳转原文链接） |
| **P0** | 已读/未读标记 | 点击文章后自动标记为已读，视觉上与未读有区别 |
| **P1** | 按订阅源筛选 | 可查看某个订阅源的所有文章 |
| **P1** | 收藏功能 | 可收藏文章，单独查看收藏列表 |
| **P2** | 全文抓取 | 对于摘要型RSS，抓取原文全文 |

---

## 三、给候选人的提示

### 3.1 时间管理建议

- **前 5 分钟**：理解需求，想清楚数据结构和前后端接口，有疑问提前问清楚
- **5-20 分钟**：完成 P0 核心功能（添加订阅、文章列表、已读标记）
- **20-30 分钟**：完成 P1 功能（筛选、收藏、刷新）
- **30-35 分钟**：测试优化，有余力可实现 P2

### 3.2 注意事项

- ⚠️ 操作后的状态变更需要持久化到后端，刷新页面不应丢失
- ⚠️ 注意空状态、边界情况的处理
